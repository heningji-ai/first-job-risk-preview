# V1.2 应届生第一份工作路径适配评分模型规格

## 1. 文档定位

本文档定义 V1.2 的底层评分模型。它是产品与开发对齐用的规格文档，不是运行时配置文件。

V1.2 不是性格测试，也不是能力考试，而是用猎头视角，提前预演一个应届生选择某类公司、某类岗位后的真实路径适配风险。

核心问题不是“你是什么人”，而是：

> 你现在选择的这类公司和这类岗位，在真实筛选、入职、适应和成长过程中，可能会在哪些地方考验你。

## 2. 判断逻辑

前面懂人：

- 学历和学校背景。
- 专业与目标方向的相关性。
- 实习、项目、作品、证书等求职材料。
- 选择这条路的动机。
- 对第一份工作的担心和预期。
- 日常工作方式底色。

后面懂职场：

- 目标公司类型的真实筛选门槛。
- 目标公司类型的日常环境。
- 目标岗位方向的高频业务场景。
- 新人会遇到的业务压力。
- 进入筛选池、入职适应和长期发展的关键卡点。

最终判断回答五个问题：

1. 能不能先进入这类公司的筛选池。
2. 为什么想走这条路，这个理由是否稳定。
3. 工作方式底色和这类公司 / 岗位是否冲突。
4. 进入真实公司和岗位场景后容易卡在哪里。
5. 最大障碍来自哪里。

## 3. 四大维度与权重

产品解释层保留四大维度。

| 产品维度 | 权重 | 说明 |
| --- | ---: | --- |
| 硬门槛 / 准入风险 | 30% | 评估用户能否进入目标公司类型的基础筛选池。 |
| 动机与预期 | 20% | 评估用户选择这条路径的理由是否稳定，预期是否与真实体验冲突。 |
| 性格底色 / 工作方式 | 30% | 评估用户的日常工作方式与目标公司、岗位要求是否存在摩擦。 |
| 情景模拟反应 | 20% | 评估用户面对真实公司和岗位场景时的反应方式。 |

工程落地时拆成五个子分数：

```txt
pathFitScoreRaw =
  admissionFitScore * 30%
  + motivationFitScore * 20%
  + baseWorkStyleFitScore * 20%
  + companyScenarioFitScore * 10%
  + roleScenarioFitScore * 20%
```

产品解释仍归为四大维度：

- `admissionFitScore` 对应“硬门槛 / 准入风险”。
- `motivationFitScore` 对应“动机与预期”。
- `baseWorkStyleFitScore + companyScenarioFitScore` 共同解释“性格底色 / 工作方式”。
- `roleScenarioFitScore` 对应“情景模拟反应”。

## 4. 维度分数区间解释

这些区间用于工程实现和内部解释，不一定直接展示给普通用户。

| 分数区间 | 内部解释 | 结果页表达方向 |
| --- | --- | --- |
| 85-100 | 明显顺手 | 这条路径和用户当前准备、预期、工作方式较一致。 |
| 70-84 | 基本可走 | 可以尝试，但需要看清真实压力点。 |
| 55-69 | 存在摩擦 | 路径里有明显需要提前验证的地方。 |
| 40-54 | 压力较大 | 进入或适应过程可能会出现较多卡点。 |
| 0-39 | 强冲突或高风险 | 当前材料、预期或工作方式与目标路径存在较强冲突。 |

## 5. 维度输入题目

| 子分数 | 主要输入题目 | 影响方式 |
| --- | --- | --- |
| admissionFitScore | A2-A6，结合 A7 目标公司类型 | 判断学历、学校/专业竞争力、专业相关性、实习与证明材料相对目标公司的筛选压力。 |
| motivationFitScore | M1-M5，结合 A7/A8 | 判断动机、担心、期待、不可接受项与目标路径的稳定性。 |
| baseWorkStyleFitScore | W1-W10，结合 A7/A8 | 判断用户工作方式底色是否与目标公司和岗位要求冲突。 |
| companyScenarioFitScore | 用户所选公司类型对应的 3 道场景题 | 判断用户面对目标公司真实日常时的适应方式。 |
| roleScenarioFitScore | 用户所选岗位方向对应的 4 道场景题 | 判断用户面对目标岗位高频业务场景时的适应方式。 |

## 6. 公司类型适配矩阵

| 公司类型 | 更看重什么 | 减分信号 | 典型新人压力 | 更适合的工作方式 | 不适合的预期 |
| --- | --- | --- | --- | --- | --- |
| 国企 / 事业单位 / 央国企平台 | 稳定履历、专业匹配、规则意识、流程耐心、长期主义 | 学历或专业门槛明显不足；无法接受流程审批；期待快速反馈和快速上升 | 流程慢、层级多、反馈不直接、成长节奏稳定但不快 | 能按规则推进、能等待、能维护关系、能长期积累 | 期待自由发挥、快速涨薪、快速证明自己 |
| 外企 / 跨国公司 | 专业表达、边界意识、英文或跨文化沟通、独立推进 | 不习惯清晰边界；等待别人手把手带；表达不清晰 | 边界清楚但陪伴少；跨团队协作；规则理解成本高 | 能主动澄清需求、按规范表达、尊重边界 | 期待强陪伴、模糊也能被照顾、所有问题都有人兜底 |
| 大厂 / 大平台 | 学习速度、数据意识、抗压、协同、结果导向 | 对高频变化敏感；怕强反馈；过度依赖明确指令 | 节奏快、竞争强、反馈直接、跨部门沟通复杂 | 能快速学习、能复盘、能承受反馈、能用结果说话 | 期待稳定慢节奏、轻松成长、只做自己喜欢的部分 |
| 创业公司 | 主动性、模糊中推进、复合能力、资源有限下解决问题 | 需要标准答案；不愿承担边界外任务；抗不确定弱 | 方向常变、一人多岗、资源少、流程不完整 | 能自驱、能补位、能边做边学、能接受变化 | 期待清晰培养体系、稳定职责、成熟流程 |
| 中小民企 / 传统民企 | 落地执行、沟通灵活、成本意识、实际问题处理 | 对管理不规范非常敏感；只接受完整培训；不愿处理琐碎问题 | 职责边界不清、管理差异大、培训少、靠实战成长 | 能和人打交道、能处理不完美、能在真实事务中学习 | 期待制度完善、流程标准、每一步都有指导 |

## 7. 岗位类型适配矩阵

| 岗位类型 | 更看重什么 | 减分信号 | 典型业务场景 | 高频压力 | 更适合的工作方式 |
| --- | --- | --- | --- | --- | --- |
| 销售 / 商务 / 客户拓展 | 主动触达、抗拒绝、跟进耐心、目标意识 | 害怕陌生沟通；被拒后投入感下降；只接受稳定反馈 | 陌生客户沟通、长期跟进、方案谈判、目标拆解 | 拒绝、业绩目标、客户变化、转化压力 | 能主动出击、能复盘话术、能持续跟进 |
| 运营 / 项目 / 用户增长 | 多方协调、细节跟进、目标拆解、数据复盘 | 不愿处理琐碎；怕临时变化；只想做想法不想落地 | 活动推进、用户反馈、数据复盘、跨部门协同 | 目标变化、细节遗漏、结果压力、协调成本 | 能跟进细节、能协调人、能对结果负责 |
| 内容 / 市场 / 品牌 | 持续产出、用户感知、表达能力、接受反馈 | 只想自由表达；难接受数据不好；不愿持续改稿 | 内容策划、投放复盘、热点响应、品牌表达 | 作品被否、数据反馈差、创意受限制、持续产出 | 能兼顾表达和反馈、能修改、能持续积累素材 |
| 技术 / 数据 / 产品 | 结构化思考、持续学习、需求理解、质量意识 | 怕需求变化；不愿长期学习；只想做确定任务 | 需求澄清、方案设计、排期、测试、数据分析 | 需求不清、专业反馈、排期压力、业务变化 | 能拆问题、能学新工具、能和业务沟通 |
| 职能支持 | 稳定细致、流程意识、服务意识、边界沟通 | 讨厌重复细节；看不到成果时投入感下降；对服务关系敏感 | 招聘、行政、财务、人事、供应链、客服支持 | 重复、低可见度、跨部门需求、流程冲突 | 能细致、能维护流程、能稳定服务内部客户 |

## 8. 封顶规则

封顶规则不是惩罚用户，而是防止后面场景题选择积极后，掩盖前面的真实路径障碍。

可执行规则：

```txt
if admissionFitScore < 45:
  finalPathFitScore = min(finalPathFitScore, 62)

if admissionFitScore < 35:
  finalPathFitScore = min(finalPathFitScore, 55)

if motivationFitScore < 45:
  finalPathFitScore = min(finalPathFitScore, 70)

if baseWorkStyleFitScore < 45:
  finalPathFitScore = min(finalPathFitScore, 65)

if companyScenarioFitScore < 45:
  finalPathFitScore = min(finalPathFitScore, 68)

if roleScenarioFitScore < 45:
  finalPathFitScore = min(finalPathFitScore, 68)
```

补充规则：

1. 同一用户触发多个封顶规则时，使用最低 cap。
2. cap 只限制最终路径适应度，不改变各维度原始分数。
3. cap 触发原因必须进入结果页解释。
4. cap 触发原因优先用于最大障碍判断。

## 9. 最大障碍判断算法

五个落地维度到结果页障碍类型的映射：

| 落地维度 | 结果页障碍类型 |
| --- | --- |
| admissionFitScore | 准入门槛 |
| motivationFitScore | 动机预期 |
| baseWorkStyleFitScore | 工作方式 |
| companyScenarioFitScore | 公司环境 |
| roleScenarioFitScore | 岗位场景 |

判断规则：

1. 先检查是否触发封顶规则。
2. 如果触发多个封顶规则，选择 cap 最低的维度作为最大障碍。
3. 如果 cap 相同，选择原始维度分数最低的维度。
4. 如果原始维度分数仍相同，按以下优先级：
   - 准入门槛
   - 工作方式
   - 动机预期
   - 岗位场景
   - 公司环境
5. 如果没有触发封顶规则，则选择五个落地维度中最低分作为最大障碍。
6. 如果最低分差距小于 5 分，可以展示“主要障碍 + 次要障碍”。
7. 最大障碍只用于结果解释，不单独替代 `finalPathFitScore`。
8. 风险卡只能作为解释补充，不能覆盖最大障碍主判断。

## 10. 最大障碍解释方向

| 最大障碍 | 结果页解释方向 |
| --- | --- |
| 准入门槛 | 现在最大的压力，可能不是入职以后能不能适应，而是这类公司在第一轮筛选时，对学历、学校、专业和相关经历要求比较高。 |
| 动机预期 | 选择这条路的理由，和这条路真实带来的工作体验可能有落差。 |
| 工作方式 | 日常工作方式和这类公司 / 岗位的要求之间存在一定摩擦。 |
| 公司环境 | 目标公司类型可能会放大用户对流程、节奏、自由度或不确定性的压力。 |
| 岗位场景 | 不是一开始就不能做，而是在该岗位真实高频场景里，可能更容易卡住。 |

## 11. 风险卡解释层规则

V1.2 中，风险卡不再决定最终结果。

风险卡使用规则：

1. 风险卡可以解释某个维度为什么低。
2. 风险卡可以用于“同时需要留意”的辅助说明。
3. 风险卡不能覆盖四维主模型。
4. 风险卡不能单独决定 `finalPathFitScore`。
5. 风险卡不能单独决定最大障碍。
6. 当风险卡与四维模型冲突时，以四维模型为主。
7. 风险卡最多展示 2 个辅助解释。
8. 风险卡解释必须转成自然语言，不显示 `cardId` / `score` / `pathFitBand`。

## 13. 结果页 V2 展示方向

结果页 V2 建议展示：

1. 路径预演总判断。
2. 路径适应度参考值。
3. 四大维度拆解：
   - 准入适配
   - 动机预期
   - 工作方式
   - 情景反应
4. 最大障碍来自哪里。
5. 用户选择的公司类型真实要求。
6. 用户选择的岗位真实场景压力。
7. 需要留意的地方。
8. “不要被一次测试定性”的边界说明。
9. 公众号关注入口。

结果页 V2 不应只依赖单一风险卡，也不应把结果包装成诊断结论。

## 14. 普通用户页禁止表达

普通用户页不得展示：

- A 档 / B 档 / C 档 / D 档
- 档位
- 评级
- 等级
- score
- cardId
- pathFitBand
- option.id
- 诊断分数
- 能力分数
- 职业匹配分
- 性格匹配度
- 你不适合
- 你就是
- 必须放弃
- 免费咨询
- 立即咨询
- 购买服务
- 保证入职
- 企业微信
- 回复【重估】

普通用户页可以展示自然语言路径判断，例如：

- 这条路和你现在的准备比较匹配。
- 这条路可以尝试，但需要先看清真实压力。
- 这类路径可能会在某些场景里放大你的适应压力。
