# RESULT_PAGE_SCENE_EXPERIENCE_NOTES

## 1. 为什么改成场景化结果页

产品方确认，未来结果页不是传统测试报告页，而是一个独立的结果体验页。

目标是让用户在一页里看到文字结果、场景氛围、未来动漫场景入口和私域承接，而不是阅读一份字段堆叠式报告。

## 2. 本阶段完成内容

本阶段将 ResultPage 调整为场景化结构：

- 场景 Hero 区
- 氛围音乐入口 UI 占位
- 路径选择展示
- 自然语言路径适配结论展示
- 路径适应度与风险预演指标展示
- 场景叙事
- 公司与岗位期待
- 可能困扰
- 默认折叠的更多解释
- 公众号 / 企业微信轻量承接

## 3. 未来目标

未来结果页可以继续接入：

- 文字结果
- 动漫场景
- 氛围音乐
- 一页式沉浸体验

本阶段没有新增真实图片资源，也没有新增真实音乐文件。

## 4. 展示层边界

`resultExperiencePresentation.ts` 只负责展示层组织，包括：

- 路径标签
- A/B/C/D 内部展示层枚举，仅用于文案、动画、音乐和视觉氛围映射
- 路径适应度参考值
- 6 个月离职风险预演值
- 30 岁后职场焦虑风险预演值
- sceneMood
- 场景文案
- 公司与岗位期待
- 时间影响表达

它不参与评分，不参与风险卡触发，不读取 triggerRules，也不读取 protectRules。

## 5. 结果页边界

普通用户页仍然只展示一个主结果。

H0 fallback 仍然有效。

普通用户页不展示内部字段、copy 状态、评分细节、触发信号或配置 key。

## 6. 本阶段未修改内容

本阶段没有修改：

- `risk_cards.json`
- `risk_card_copy.json`
- 任何其他 JSON 配置
- scoringEngine
- riskCardEngine
- resultPipeline
- validate-config
- test-risk-logic

本阶段也没有新增后端、数据库、openid、登录、支付、表单、真实分享链路、真实音乐文件或真实图片资源。
