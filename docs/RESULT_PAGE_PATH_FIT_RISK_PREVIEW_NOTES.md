# RESULT_PAGE_PATH_FIT_RISK_PREVIEW_NOTES

## 1. 为什么升级为路径适配风险预演

产品方确认，结果页的核心问题不是“给用户一个解决方案”，也不是“做职业诊断”，而是回答：

用户选择的这类公司 + 这类岗位，和当前性格、预期、承压方式之间，有多大摩擦。

因此本阶段将结果页从“路径判断”升级为“路径适配风险预演”。

## 2. 表达边界

页面不说“你不适合这个岗位”。

页面使用更克制的表达，例如：

- 这个岗位未必适合你现在的性格和工作预期
- 这条路对你当前阶段不太友好
- 这类岗位要求的工作方式，和你更舒服、更容易发挥的环境之间存在差距

## 3. A/B/C/D 内部枚举含义

A/B/C/D 只作为展示层内部枚举，用于选择固定文案块、动画、音乐和视觉氛围，不在普通用户页直接展示。

- A：路径适应度较高，可以明确鼓励用户大胆尝试。
- B：基本可走，但不要只看表面。
- C：适配摩擦明显，需要看清 3 到 6 个月内的适应成本。
- D：当前阶段不太友好，可能放大当前最敏感的压力点。

普通用户看到的是自然语言路径判断，不是考试、个人能力评价或被打低分。

高匹配时需要明确恭喜用户，因为这代表当前路径未出现强拦路信号。

## 4. 三个结果指标

结果页展示三个预演指标：

- 路径适应度参考值
- 6 个月离职风险预演值
- 30 岁后职场焦虑风险预演值

路径适应度参考值不是个人能力评价，也不是职业诊断结论。

6 个月离职风险预演值不是统计口径下的真实发生比例，也不是对个人未来行为的确定预测。

30 岁后职场焦虑风险预演值用于表达长期路径代价，不代表未来一定发生。

## 5. 不讲解决方案

本阶段结果页只提示风险可能性，不讲解决方案。

普通用户页不展示：

- 应该换到什么岗位
- 应该去什么方向
- 应该怎么解决
- 应该怎么改简历
- 应该怎么准备面试
- 风险降低做法
- 找谁验证
- 猎头季哥可以帮你看什么
- 验证清单

## 6. 动画和音乐承载

A/B/C/D 内部枚举未来可以分别接入不同动画和音乐。

本阶段只保留展示层字段：

- `pathFitBand`
- `sceneMood`
- `animationPreset`
- `musicMoodKey`
- `musicMoodLabel`

本阶段只使用 CSS 轻动效，没有新增真实图片资源，没有新增真实音乐文件，没有引入 Lottie 或第三方动画库。

## 7. 配置和逻辑边界

本阶段没有修改任何 JSON 配置。

本阶段没有修改评分逻辑、风险卡触发逻辑或 resultPipeline。

`resultExperiencePresentation.ts` 只负责展示层映射，不参与评分和风险触发。
