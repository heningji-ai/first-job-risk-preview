# 正式结果页文案模板

本文件是正式结果页文案模板。请勿在模板阶段生成正式用户结论。

禁止文案方向：

- 不写“你就是某某风险型”。
- 不写未经确认的职业诊断。
- 不写正式建议用户不要选择某类工作。
- 不把工程示例卡包装成正式判断。

## 1. 顶部结果标题模板

填写项：

- 页面主标题：待产品方填写
- 页面副标题：待产品方填写
- 工程或免责声明是否保留：待产品方确认
- 适用人群：待产品方填写

模板：

```txt
标题：待填写
副标题：待填写
边界说明：待填写
```

## 2. 风险卡展示文案模板

每张卡填写：

- riskCardId：待填写
- 卡片标题：待填写
- 一句话解释：待填写
- 触发摘要：待填写
- 用户可理解的例子：待填写
- 不应触发时的替代表达：待填写

模板占位：

```txt
本卡展示文案待产品方填写。
```

注意：以上只是占位，不是正式结论。

## 3. 触发解释文案模板

填写项：

- answer 命中解释：待填写
- dimension 命中解释：待填写
- finalRisk 命中解释：待填写
- flag 辅助解释：待填写
- protectRules 跳过解释：待填写

模板占位：

```txt
本卡进入预览，是因为若干工程信号命中。正式解释需由产品方补充。
```

## 4. 建议行动文案模板

填写项：

- 低压力行动：待填写
- 进一步自查行动：待填写
- 求职准备行动：待填写
- 需要人工服务承接的行动：待填写

边界：

- 不给绝对化职业建议。
- 不使用恐吓式表达。
- 不把测评当成诊断。

## 5. 低风险 fallback 文案模板

用于 `H0_GENERAL_REMINDER`：

- fallback 标题：待填写
- fallback 解释：待填写
- fallback 行动建议：待填写
- fallback 分享文案：待填写

要求：

- H0 只作为展示兜底。
- H0 不进入 `triggeredRiskCards`。

## 6. viral copy 模板

默认 viral copy：

```txt
targetText：待填写
copyText：待填写
```

风险卡专属 viral copy：

| riskCardId | targetText | copyText | 是否正式确认 |
| --- | --- | --- | --- |
| 待填写 | 待填写 | 待填写 | 待填写 |

注意：

- 缺少专属 viral copy 时使用 `defaultViralCopy`。
- `defaultViralCopy` 缺失应视为配置错误。

## 7. 服务承接文案模板

填写项：

- 服务区标题：待填写
- 服务区说明：待填写
- 服务卡名称：待填写
- 服务卡条目：待填写
- 服务边界：待填写
- 不承诺事项：待填写

模板占位：

```txt
服务承接说明待产品方填写。具体服务内容和边界需由产品方确认。
```

注意：

- 第一版只展示服务信息。
- 不接支付。
- 不接预约。
- 不接数据库或外部服务。
