# V1.1 Student Test Experience Notes

## 本阶段完成内容

本阶段继续修正应届生答题前和答题过程的体验，不调整结果页主判断逻辑。

主要改动：

- 首页从功能入口改为更能回应第一次找工作迷茫感的入口。
- 首页补充猎头季哥 21 年招聘和猎头经验与本测试之间的关系。
- 首页增加轻量视觉 Hero 区，让页面不再只有文字。
- 答题开场改为独立引导页，从“找工作以你为主”开始，让用户先感到被理解。
- A5、A6、A8 的题干改成更贴近应届生真实感受的表达。
- A9 继续保留在配置中，但普通用户答题时不再展示，系统默认按“不知道 / 不确定”进入快速倾向题。
- 答题过程用 5 张章节过渡卡替代题目顶部长说明。
- 最后一题后先进入更自然的过渡卡，再由用户点击进入结果页。

## 为什么首页需要增强信任感

只说“21 年经验”并不能自然建立信任。应届生更需要知道：这段经验为什么和自己的第一份工作选择有关。

因此首页补充说明：猎头季哥长期同时观察企业怎么筛人、怎么用人，也观察很多年轻人在第一份工作中为什么迷茫、痛苦和怀疑自己。

这样能让用户理解，这个测试不是泛泛提问，而是把招聘一线看到的用人标准、岗位日常和新人适应风险整理成问题。

## 为什么首页增加视觉 Hero 区

首页如果只有文字，会让第一次进入的用户感到信息密度偏高，也不容易形成“站在选择路口”的感受。

因此本阶段用 CSS 做了轻量视觉 Hero 区：一个抽象人物、几条分叉路径和几张漂浮卡片，帮助承载公司类型、岗位方向和第一份工作的选择感。

当前视觉区不加载真实图片，也不引用外部资源。未来如果产品视觉方向确定，可以替换为真实动漫插画。

## 为什么首页要更理解第一次找工作的迷茫

很多应届生不是不努力，而是第一次面对公司类型、岗位名称、父母建议、同学选择和现实压力时，不知道该相信什么。

首页需要先承认这种迷茫是正常的，再说明本测试只是帮助用户提前看见路径适配风险，不替用户决定人生。

## 为什么 current_status 前也需要引导页

找工作不是先把用户塞进一个岗位名里。

真正每天面对工作方式、节奏、反馈和压力的人，是用户自己。

因此第一个问题前增加独立引导页，先让用户理解：这次不是先从岗位开始，而是先从自己现在的位置开始。

这张卡不算题目、不保存答案，也不进入进度统计。

## 为什么 company_type / work_type 前需要独立引导页

公司类型不只是名气不同，背后也包括日常节奏、管理方式和新人空间。

岗位也不只是名称不同，背后是每天反复面对的工作方式和要求。

因此 company_type 和 work_type 前各增加一张独立引导页，让用户先理解为什么要选这两个方向，再进入具体题目。

进入题目后，题目页只保留小标题、题目和选项，避免用户一边读长说明一边做选择。

## 为什么担心问题题删除长辅助说明

担心问题本身已经足够清楚。

如果再在题目页顶部放一段心理说明，会让页面显得拥挤，也容易让用户觉得自己正在被解释。

因此本阶段删除该题长辅助说明，只保留题目和选项。

## 为什么要建立性格基准线

在进入更具体的情景前，需要先看用户在压力、反馈、不确定和沟通里通常会怎么反应。

这能帮助后续判断公司环境、岗位要求和用户当前状态之间是否容易适配。

这张卡出现在快速倾向问题前，只解释接下来为什么要问这些基础反应。

## 为什么要增加情景模拟过渡页

用户完成前面的基础选择和反应问题后，需要知道接下来会进入更贴近第一份工作的真实场景。

因此在新人阶段相关问题前增加一张过渡卡，说明后面会模拟没人一步步告诉你怎么做、标准不清、反馈不及时等情况。

## 为什么停用后段重复过渡页

公司类型和岗位方向已经在前面各自做了独立引导。

后面如果再次解释同一层意思，会打断答题节奏，也会让用户觉得流程变长。

因此本阶段停用后段两张重复过渡页，相关题目仍然正常出现，分流和答案保存不受影响。

## 章节过渡卡边界

章节过渡卡只用于解释“为什么接下来要问这些问题”。

它们不算题目、不进入题号、不进入进度条、不保存答案，也不影响 localStorage session、评分和风险触发。

点击继续后，用户会进入对应的真实题目。

## A9 处理方式

A9 的 mbti_known 继续留在 questions.json 中，作为配置结构的一部分。

普通用户答题时不再展示 A9，系统默认记录 mbti_known 为 unknown，并继续进入快速倾向题。

这样做是因为 B1 / mbti_type 仍未补齐，第一版不让用户在这里被打断。

## 最后一题后的过渡卡

完成最后一题后，系统会先保存 localStorage session，再展示过渡卡。

用户点击“查看我的预演结果”后进入结果页。

这个过渡只改善体验，不新增后端、不新增接口、不改变结果页计算。

## 后续可继续优化

后续可以继续通过手工浏览器验收观察：

- 首页是否足够有信任感。
- 答题页提示是否像对话，而不是系统说明。
- 最后一题后的过渡卡是否自然。
- A9 隐藏后快速倾向题是否衔接自然。
